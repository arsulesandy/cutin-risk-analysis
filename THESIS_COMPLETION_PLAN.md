# Thesis Completion Plan (2 Weeks)

This plan is the minimum practical path to make the thesis package complete, reproducible, and defensible.

## Final Definition Of Done

All items below should be true:

- Main claims are mapped to clear metrics, figures, and tables.
- Validation protocol is leakage-safe (recording-level split, no train/test mixing).
- Core results include uncertainty (95% CI).
- Baselines and ablations are reported.
- Failure cases are analyzed.
- Reproducibility is one-command and documented.
- Core pipeline logic has tests for critical pieces.

## Workstreams

1. Claim-to-evidence matrix.
2. Split integrity and leakage checks.
3. Confidence intervals and statistical robustness.
4. Baselines and ablations.
5. Error analysis (false positives / false negatives).
6. Calibration and threshold operating points.
7. Reproducibility hardening.
8. Test coverage and thesis-ready documentation.
